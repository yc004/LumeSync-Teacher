# Contributing Guide

## Development Flow

1. Create a feature branch from `main`.
2. Keep changes scoped and focused.
3. Run local checks before opening a PR.
4. Open a PR with clear context, testing steps, and risk notes.

## Commit Guidelines

- Use clear commit messages.
- Separate refactors from behavior changes when possible.
- Avoid unrelated formatting-only churn.

## Pull Request Checklist

- Code builds locally.
- Relevant tests pass.
- Docs updated when behavior or APIs changed.
- Backward compatibility impacts are documented.
